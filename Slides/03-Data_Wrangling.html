Data Wrangling with the Tidyverse

The tidyverse

  • One philosophy of the tidyverse is building pipe-friendly functions.

  • Another philosophy is using English verbs as function names instead of symbols or abbreviations.

  • Some consider this more accessible/readable

penguins |>
  filter(species == "Adelie")
penguins[penguins$species == "Adelie",]

The tidyverse

Pros:

  • Fairly consistent behavior of functions

    • Predictable input and output types
    • “opinionated”

  • Semi-guaranteed maintenance of packages

(because Posit, PBC pays developers!)

  • More available and polished documentation and tutorials

  • Welcoming community

The tidyverse

Cons:

  • Sometimes less flexible for complex tasks

  • For big data, there are faster options

  • Updates frequently -> sometimes this makes your old code unrunnable.

  • Some people object to Posit’s strong influence over an open-source community.

Should you use the tidyverse???

Even the most die-hard tidyverse fanatics use base R and other packages quite frequently.

Above all, the best code approach is the one that

  • you enjoy the most

  • gets job done

  • lets you collaborate with colleagues.

Using mutate to clean data

One common use of mutate is to make sure our data types are correct.

Most often, this involves turning character variables into factor variables:

penguins <- penguins |>
          mutate(
            species = factor(species)
          )

case_when

A common use of mutate() is to turn a quantitative variable into a categorical one

case_when makes this process easier.

penguins |>
  mutate(
    size = case_when(
      body_mass_g < 3000 ~ "Tiny",
      body_mass_g < 5000 ~ "Medium",
      body_mass_g < 7000 ~ "Whomper"
    )
  )

Grouping and summarizing

Verb: group_by

Before we do something like summarize, we group_by a categorical variable.

penguins |>
  group_by(species) |>
  summarize(mean(body_mass_g))
# A tibble: 3 × 2
  species   `mean(body_mass_g)`
  <fct>                   <dbl>
1 Adelie                  3706.
2 Chinstrap               3733.
3 Gentoo                  5092.

Warning!

You should always have a summarize after a group_by!!!

(There are other ways to use it, but wait until you are very familiar with dplyr to try that.)
